# Architecture Decision Records (ADRs)

Lightweight log of significant technical and architectural decisions for Ankify.

One file per decision. Decisions are **immutable once shipped** — if we reverse a decision, write a new ADR that supersedes the old one (don't edit the original).

## Naming convention

```
ADR-<number>-<short-slug>.md
```

## Statuses

| Status | Meaning |
|--------|---------|
| `accepted` | Decision is in effect |
| `superseded` | Replaced by a newer ADR (link to it) |
| `proposed` | Under discussion, not yet committed |

## Creating a new ADR

Copy `_TEMPLATE.md`. Keep it short — a good ADR is one page, not a whitepaper.
